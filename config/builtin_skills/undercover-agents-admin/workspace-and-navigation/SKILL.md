# Workspace and Navigation

Use this skill for questions about where things live in the admin area and how the day-to-day workspace is structured.

## Main ideas

- The admin app is organized around the current operation. The selected operation decides which agents, missions, tools, skill catalogs, and RAG flows the user is working with.
- Every tenant also has a Headquarter operation for shipped builtins and shared internal setup, plus a Default operation for normal working content.
- Most admin pages follow the same shape: primary actions in the page header, the main surface in the center, and a shared right-side panel for Agent Alpha and page-specific side panels.

## How users move around

- Use the operation switcher when records seem to be missing or the user needs a different workspace.
- Use the left navigation to move between resource areas and the right panel when the user needs in-page help or a page-specific side workflow.
- Treat the current page and current operation as the first two facts to confirm before giving navigation guidance.

## Practical guidance

- If a built-in admin surface is missing, Headquarter is usually the right operation to check.
- If a day-to-day record is missing, the user is often in the wrong operation rather than looking at deleted data.
- When explaining a workflow, describe both where the main page lives and what the right panel contributes on that page.
