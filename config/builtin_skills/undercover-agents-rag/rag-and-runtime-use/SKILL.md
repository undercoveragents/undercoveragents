# RAG and Runtime Use

Use this skill for questions about how prepared retrieval knowledge becomes useful in real conversations or workflows.

## How RAG supports runtime behavior

- A prepared RAG flow becomes valuable when its stored knowledge is exposed through runtime surfaces that can answer grounded questions.
- Retrieval is most useful when it is attached to a clear user need, such as factual lookup, policy recall, or document-aware assistance.
- Good RAG usage depends on both strong source preparation and clear runtime assignment.

## Practical guidance

- If answers are not grounded enough, review both the knowledge pipeline and the consuming runtime surface.
- If the user wants the system to know a body of documents, decide whether that knowledge should be modeled as a skill, a RAG flow, or both.
- Explain runtime retrieval as the last mile of a larger knowledge-preparation process.
