# Client Channels and Preview

Use this skill for questions about branded end-user chat experiences.

## What client channels control

- Client channels control the public chat presentation for an operation.
- They own branding-oriented content such as title, welcome message, footer, and user-facing labels.
- A client channel also chooses the primary agent that handles that end-user conversation.

## Preview and default behavior

- The admin UI includes a live preview for client channels so operators can review branding and copy before publishing changes.
- One client channel can act as the default channel for the operation's public chat entry point.
- Preview work is about the end-user experience, not about changing the underlying agent instructions unless the requested behavior truly belongs there.

## Practical guidance

- If the user wants to change wording, labels, or branded framing, keep the work on the client channel first.
- If the preview looks wrong, confirm the current channel settings before assuming the assigned agent is at fault.
- Explain the default flag as the operation's primary public entry point, not as a cross-tenant setting.
