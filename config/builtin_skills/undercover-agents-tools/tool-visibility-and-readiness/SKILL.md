# Tool Visibility and Readiness

Use this skill for questions about why a tool is not yet available or why it is only partially exposed.

## Readiness model

- Many tools require setup work before they are truly ready to help an agent.
- That preparation can include discovery, validation, analysis, or a visibility decision about what the tool should expose.
- A tool that exists in the database is not always a tool that is safe or useful at runtime.

## Visibility

- Visibility settings decide what parts of a tool's underlying data or actions should be surfaced.
- Good visibility choices keep the tool useful while still respecting the intended operator boundaries.

## Practical guidance

- If a tool does not appear usable, verify readiness before reassigning it.
- If a user wants narrower exposure, guide them toward visibility controls rather than duplicating the whole tool.
- Explain partial visibility as intentional product behavior, not as a missing record.
